SHOW STATUSÂ¶

Show MemSQL server status information.

Syntax

SHOW STATUS

Notes

  • Use SHOW STATUS EXTENDED instead for detailed MemSQL-specific memory and performance statistics.
  • This command can be run on any MemSQL node (see Node Requirements for MemSQL Commands).

Output

See System Status.

Example

memsql> SHOW STATUS;
+-------------------------------------+---------+
| Variable_name                       | Value   |
+-------------------------------------+---------+
| Aborted_clients                     | 0       |
| Aborted_connects                    | 0       |
| Bytes_received                      | 177     |
| Bytes_sent                          | 1881    |
| Connections                         | 11      |
| Max_used_connections                | 2       |
| Queries                             | 77      |
| Questions                           | 5       |
| Threads_connected                   | 2       |
| Threads_created                     | 10      |
| Threads_running                     | 1       |
| Uptime                              | 11234   |
| Seconds_until_expiration            | 2539253 |
| Query_compilations                  | 0       |
| Query_compilation_failures          | 0       |
| GCed_versions_last_sweep            | 0       |
| Average_garbage_collection_duration | 0 ms    |
| Total_server_memory                 | 81.6 MB |
| Alloc_variable_cached_buffers       | 0.0 MB  |
+-------------------------------------+---------+
19 rows in set (0.00 sec)
