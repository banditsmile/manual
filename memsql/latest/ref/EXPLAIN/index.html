EXPLAINÂ¶

Show the indexes used and some hints about the query plan used to execute the query.

Syntax

EXPLAIN [EXTENDED | FORMAT=JSON] SELECT select_options

Notes

  • This command displays only one possible execution plan (except with the FORMAT=JSON option). A different plan may be selected at runtime depending on the table statistics at query runtime.
  • This command displays the plan we would generate if the query were reoptimized and recompiled now. If the query has already been run and a plan has already been saved in the plancache, that plan will be reused, even though it may differ from the plan displayed by EXPLAIN. You may run DROP ... FROM PLANCACHE to force the query to be reoptimized and recompiled.
  • When run on an aggregator, the EXTENDED option displays additional details, such as the queries executed on the leaves.
  • When run on a leaf, the EXTENDED option uses the rows column to give an estimate on the number of rows that need to be scanned to run the query. It also displays the plan that would be selected based on the current table statistics. The EXTENDED option is a lot more expensive because it performs code-generation.
  • The FORMAT=JSON option gives more detailed output as JSON.
  • select_options are any options specified in the * SELECT statement.
  • This command can be run on any MemSQL node (see Node Requirements for MemSQL Commands).

Examples

memsql> EXPLAIN SELECT * FROM mytbl;
+----+-------------+-------+------+---------------+------+---------+------+------+-------+
| id | select_type | table | type | possible_keys | key  | key_len | ref  | rows | Extra |
+----+-------------+-------+------+---------------+------+---------+------+------+-------+
|  1 | SIMPLE      | mytbl | ALL  | NULL          | NULL | NULL    | NULL | NULL |       |
+----+-------------+-------+------+---------------+------+---------+------+------+-------+
1 row in set (0.00 sec)


memsql> EXPLAIN EXTENDED SELECT * FROM mytbl;
+----+-------------+-------+------+---------------+------+---------+------+------+----------+-------+
| id | select_type | table | type | possible_keys | key  | key_len | ref  | rows | filtered | Extra |
+----+-------------+-------+------+---------------+------+---------+------+------+----------+-------+
|  1 | SIMPLE      | mytbl | ALL  | NULL          | NULL | NULL    | NULL |    0 |     NULL |       |
+----+-------------+-------+------+---------------+------+---------+------+------+----------+-------+
1 row in set (1.12 sec)
