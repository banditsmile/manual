start(事件类型:sortstart)
回调函数参数:
event
原生的浏览器事件.
ui
JQuery的ui对象
详细:
排序开始, 也就是开始拖动其中的一项元素.
示例:
初始化时设置
$('.selector').sortable({ start: function(event, ui) { ... } });
动态绑定事件
$('.selector').bind('sortstart', function(event, ui) { ... });
sort(事件类型:sort)
回调函数参数:
event
原生的浏览器事件.
ui
JQuery的ui对象
详细:
排序过程, 也就是拖动sortable的项每1像素触发一次.
示例:
初始化时设置
$('.selector').sortable({ sort: function(event, ui) { ... } });
动态绑定事件
$('.selector').bind('sort', function(event, ui) { ... });
change(事件类型:sortchange)
回调函数参数:
event
原生的浏览器事件.
ui
JQuery的ui对象
详细:
顺序改变一次, 触发一次.
译者注: 需要注意的是这里并不指一个成功的改变顺序, 而是在拖动过程中, 导致ui认为如果此时松开鼠标会导致顺序发生改变, 那么就会触发一次这个事件. 可以参考sortable的tolerance选项.
示例:
初始化时设置
$('.selector').sortable({ change: function(event, ui) { ... } });
动态绑定事件
$('.selector').bind('sortchange', function(event, ui) { ... });
beforeStop(事件类型:sortbeforeStop)
回调函数参数:
event
原生的浏览器事件.
ui
JQuery的ui对象
详细:
排序结束时触发, 需要保证此时placeholder和helper有效.
译者注:
下面是处理beforeStop和stop事件的源代码:
this._trigger("beforeStop", event, this._uiHash());
for (var i=0; i < delayedTriggers.length; i++) {
delayedTriggers[i].call(this, event);
};
this._trigger("stop", event, this._uiHash());
可以看出在处理完beforeStop事件之后, 还需要处理一些延迟的事件, 比如helper平滑的回归到新位置(参见sortable的revert选项), 所以, 必须保证placeholder和helper仍然是有效的. 而在处理stop事件的时候, 则没有此要求.
示例:
初始化时设置
$('.selector').sortable({ beforeStop: function(event, ui) { ... } });
动态绑定事件
$('.selector').bind('sortbeforeStop', function(event, ui) { ... });

receive(事件类型:sortreceive)
回调函数参数:
event
原生的浏览器事件.
ui
JQuery的ui对象
详细:
被连接的sortable从其他sortable(连接它的sortable中的一个)接受到一个元素之后, 触发这个事件.
译者注:
关于连接请参见sortable的connectWith选项.
这里这个事件只有在鼠标松开, 一个元素真正进入的时候, 才触发.
示例:
初始化时设置
$('.selector').sortable({ receive: function(event, ui) { ... } });
动态绑定事件
$('.selector').bind('sortreceive', function(event, ui) { ... });

over(事件类型:sortover)
回调函数参数:
event
原生的浏览器事件.
ui
JQuery的ui对象
详细:
一个元素移动到连接到sortable上方时, 触发此事件.
译者注: 此事件需要两个sortable联动, 详见示例3
示例:
初始化时设置
$('.selector').sortable({ over: function(event, ui) { ... } });
动态绑定事件
$('.selector').bind('sortover', function(event, ui) { ... });
译者注:
1. 事件定义到被连接的sortable中. 2. 连接方的tolerance选项影响ui对over的判定 3. 此事件不需要松开鼠标, 每次over时都会触发.
显示演示示例
  • 可排序元素1
  • 可排序元素2
    • <style type="text/css">
    #sortable{ list-style-type: none; margin: 0; padding: 0; width: 200px;}
    #sortable>li{height: 30px;}
    #receiver{width: 300px; height: 300px; border: 1px solid red;}
    </style>
    <script type="text/javascript">
    $(function() {
    $("#sortable").sortable({
    tolerance: 'pointer',
    connectWith: '#receiver'
    });
    $('#receiver').sortable({
    over: function(event, ui) {
    $('#msg').html($('#msg').html() + "Helloworld<br />");
    }
    });
    });
    </script>
