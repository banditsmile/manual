可选类型:
选择器
DOM元素
jQuery封装元素
布尔值
设置false表示刚开始全部关闭. 译者注: 官方文档说需要设置collapsible: true, 经过测试不需要.
数字
索引值, 也就是第几个. 这里从0开始计算.
描述:
指初始化后默认打开哪个header对应的内容面板.
译者注: 下面是处理这个值的源码
return selector
? (typeof selector == "number"
? this.headers.filter(":eq(" + selector + ")")
: this.headers.not(this.headers.not(selector)))
: (selector === false
? $([])
: this.headers.filter(":eq(0)"));
首先, 判断了selector(这里的selector就是我们设置的active的值)的布尔值
1. selector布尔值为真: 判断selector的类型是否为数字
1.1 selector是数字类型: 选择数字指定的索引对应元素
1.2 selector不是数字类型: 这里是这段代码的精华, 用两个not对this.headers进行了过滤, 不仅处理了将字符串类型的选择器或普通DOM元素转换成jQuery对象, 而且, 由两次not从headers中将设定的header元素找了出来.
2. selector布尔值为假: 判断selector的值是否是false, 注意, 这里是真正的值而不是布尔值
2.1 selector为false: 设置没有活动的内容面板
2.2 selector不为false(通常是未设置): 这里就是默认情况, 将第0个设置成了默认活动的.

$('.selector').accordion({ active: 2 });
获取选项值
var active = $('.selector').accordion('option', 'active');
设置选项值
$('.selector').accordion('option', 'active', 2);

描述:

译者注:
下面是这个属性处理的源码:
if(obj.left != undefined) this.offset.click.left = obj.left + this.margins.left;
if(obj.right != undefined) this.offset.click.left = this.helperProportions.width - obj.right + this.margins.left;
if(obj.top != undefined) this.offset.click.top = obj.top + this.margins.top;
if(obj.bottom != undefined) this.offset.click.top = this.helperProportions.height - obj.bottom + this.margins.top;
obj就是我们设置的对象, 可以看出, 是按照先左后右, 先上后下的顺序进行设置的, 因此, 如果即设置了left, 又设置了right, 那么right会覆盖left.

    以下为处理forcePlaceholderSize选项的源代码:
    // 1. If a className is set as 'placeholder option, we don't force sizes - the class is responsible for that
    // 2. The option 'forcePlaceholderSize can be enabled to force it even if a class name is specified
    if(className && !o.forcePlaceholderSize) return;

    //If the element doesn't have a actual height by itself (without styles coming from a stylesheet), it receives the inline height from the dragged item
    if(!p.height()) { p.height(self.currentItem.innerHeight() - parseInt(self.currentItem.css('paddingTop')||0, 10) - parseInt(self.currentItem.css('paddingBottom')||0, 10)); };
    if(!p.width()) { p.width(self.currentItem.innerWidth() - parseInt(self.currentItem.css('paddingLeft')||0, 10) - parseInt(self.currentItem.css('paddingRight')||0, 10)); };
    这段代码在if(className && !o.forcePlaceholderSize) return;这一句中判断了没有设置className(就是指placeholder选项设置的样式名称)或者设置了forcePlaceholderSize选项为true时, 继续向下执行将placeholder的大小设置为当前拖拽的accordion元素的大小.

    helper(默认值:original)
    可选类型:
    字符串
    可选值:
    original: 原始元素, 也就是直接拖动accordion元素. 如果helper是原始元素拖动结束后不会被移除.
    clone: 拖动原始元素的一个克隆.
    函数
    函数接受两个参数, 第一个参数是整个accordion对象(jquery封装后的对象), 第二个参数是一个数组, 有两个元素, 第一个元素是jquery包装的事件, 第二个元素是当前拖拽的元素. 下面是传递函数时的相关处理源码:
    $(o.helper.apply(this.element[0], [event, this.currentItem]))
    描述:
    拖动过程中, 跟随鼠标移动的帮助用户知道当前拖动到什么位置的一个控件.
    译者注:
    下面是关于helper的处理源码.
    var helper = $.isFunction(o.helper) ?
    $(o.helper.apply(this.element[0], [event, this.currentItem])) :
    (o.helper == 'clone' ?
    this.currentItem.clone() :
    this.currentItem);
    首先判断是否是函数, 然后判断是否是clone, 如果不是以上两种, 直接返回自身.
    另下面是移除helper的相关源码:
    if(this.options.helper != "original" && this.helper && this.helper[0].parentNode) this.helper.remove();
    可以看出, 在设置为original时, helper不会被移除, 其他所有情况下, 拖动结束后, helper对象都会被移除. 所以, 在做相关处理时, 需要注意这一点. 如果使用页面已有的元素返回, 第二次拖动时, 实际上就没有那个helper可以返回给你了.
    建议: 在使用函数方式时, 请确保每次都是新创建一个元素返回作为helper.
    示例:
    初始化时设置
    $('.selector').accordion({ helper: 'clone' });
    获取选项值
    var helper = $('.selector').accordion('option', 'helper');
    设置选项值
    $('.selector').accordion('option', 'helper', 'clone');
