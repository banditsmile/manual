accept(默认值:*)
可选类型:
选择器
所有匹配指定选择器的droggable组件都可以被接收.
函数
如果指定的是一个函数, 该函数将在每个droggable组件被放入时被调用, droggable组件被作为第一个参数传入, 如果函数处理返回结果是true, 那么就认为该组件是可接受的. 其实就是一个filter过滤函数.
描述:
用来设置可以接收的droggable元素的选择器或过滤器.
示例:
初始化时设置
$('.selector').droppable({ accept: '.special' });
获取选项值
var accept = $('.selector').droppable('option', 'accept');
设置选项值
$('.selector').droppable('option', 'accept', '.special');
activeClass(默认值:false)
可选类型:
字符串
描述:
指定一个样式的名称, 在所有当前droppable控件可接收的draggable控件中任意一个拖动的时候, 就将该droppable控件的样式改变为这里设置的样式. 相当于用于提示用户, 现在正在拖动的控件可以被放到什么地方.
示例:
初始化时设置
$('.selector').draggable({ appendTo: 'body' });
获取选项值
var activeClass = $('.selector').droppable('option', 'activeClass');
设置选项值
$('.selector').droppable('option', 'activeClass', '.ui-state-highlight');
译者注: demo中, 设置了两个draggable和两个droppable, 由于设置了对应的scope, 所以只能对号入座, 因此, 拖动draggable_a的时候, droppable_a就相应的显示了设定的activeClass

addClasses(默认值:true)
可选类型:
布尔值
描述:
用来设置是否给droppable元素通过ui-droppable样式才装饰它. 主要为了在通过.droppable()初始化很多(成百个)元素的时候优化性能考虑
true表示ui-droppable样式被添加到该元素.
false表示ui-droppable样式不被添加到该元素.
示例:
初始化时设置
$('.selector').droppable({ addClasses: false });
获取选项值
var addClasses = $('.selector').droppable('option', 'addClasses');
设置选项值
$('.selector').droppable('option', 'addClasses', false);
greedy(默认值:false)
可选类型:
布尔值
如果设置为true, 在嵌套droppable时会阻止事件向上传播.
描述:
由于javascript是把HTML作为一棵DOM树来解析的, 所以, 元素之间存在父子关系. 当两个droppable控件有父子关系(只要是直系亲属关系都会发生)时, 事件会自动向上传播, 因此, 在有些情况下, 需要显式的来停止这种事件的传播.
该选项只影响drop事件, 其他事件经过测试不会传播.
事件传播是合理的, 但是在有些情况下是不需要的或会引发混乱, 比如: div中包含了一个a标签, 那么点击a的时候, 实际上也是点击了div, 因为a在它中间嘛.
示例:
初始化时设置
$('.selector').droppable({ greedy: true });
获取选项值
var greedy = $('.selector').droppable('option', 'greedy');
设置选项值
$('.selector').droppable('option', 'greedy', true);
译者注: greedy_draggable拖动到droppable_child中的时候, droppable_child是会响应事件的, 但是由于droppable_child中设置了greedy为true, droppable_parent不会响应事件

hoverClass(默认值:false)
可选类型:
字符串
描述:
通过指定一个样式名, 来给定当一个当前droppable控件可接受的draggable控件移动到它上面的时候, 使用的样式.
示例:
初始化时设置
$('.selector').droppable({ hoverClass: 'drophover' });
获取选项值
var hoverClass = $('.selector').droppable('option', 'hoverClass');
设置选项值
$('.selector').droppable('option', 'hoverClass', 'drophover');
scope(默认值:default)
可选类型:
字符串
描述:
该选项描述一个范围, 只有那些scope选项值和这里的scope选项值相同的draggable控件才可以被放到这个droppable中. droppable的accept选项也接受这个选项的约束, accept选项指定的选择器是在scope进行过滤之后进行选择的.
例如:
$('#draggable_a').draggable({scope: 'a'});
$('#draggable_b').draggable({scope: 'b'});
$('#droppable_a').droppable({scope: 'a'});
$('#droppable_b').droppable({scope: 'b'});
droppable控件的accept选项默认是'*', 看起来数draggable_a, draggable_b可以自由的放入到droppable_a和droppable_b中, 但是, 由于scope的约束, draggable_a只能放入到droppable_a, draggable_b只能发乳到droppable_b中.
注意: 这个选项就和变量的名称空间的意义类似. 默认值是'default', 说明如果不指定, 大家都还是有scope的, 名字是default而已.
示例:
初始化时设置
$('.selector').droppable({ scope: 'tasks' });
获取选项值
var scope = $('.selector').droppable('option', 'scope');
设置选项值
$('.selector').droppable('option', 'scope', 'tasks');
tolerance(默认值:intersect)
可选类型:
字符串
fit: draggable完全进入droppable
intersect: draggable至少50%进入droppable
pointer: 鼠标指针进入droppable
touch: draggable进入droppable任意多
描述:
描述droppable把怎么样一种情况认为是可接受draggable在自己上面了(主要影响的是hover, 和over相关的事件)
示例:
初始化时设置
$('.selector').droppable({ tolerance: 'fit' });
获取选项值
var tolerance = $('.selector').droppable('option', 'tolerance');
设置选项值
$('.selector').droppable('option', 'tolerance', 'fit');
